# Frontend Refactor Design Spec: Modern Trading UI

**Date**: 2026-04-29
**Topic**: Refactoring the `lobster_quant` Streamlit frontend for readability, aesthetics, and dynamic theme switching.

## 1. Vision & Purpose
To transform the current disparate and hardcoded UI into a professional-grade "Modern Trading" dashboard that defaults to a clean light mode but offers a high-contrast dark mode toggle. The UI should be consistent, readable, and unified across all tabs.

## 2. Approach & Architecture
We will use a **Centralized UI Management** pattern to separate appearance from logic.

### 2.1 Theme Management
- **Runtime Switching**: Leverage `st.set_theme()` (Streamlit 2025/2026 API) for per-session theme updates.
- **Session State**: Store `st.session_state.theme` ('light' or 'dark').
- **Unification**: Replace hardcoded CSS in `quant_tool_page.py` with dynamic CSS generated by a shared helper.

### 2.2 Visual Style: Modern Trading (Minimal)
- **Palette**: 
  - **Light**: Pure white background, slate text, thin gray borders, vibrant accent colors (Green/Red).
  - **Dark**: Deep navy/charcoal background (`#0e1117`), white/off-white text, vibrant accents.
- **Components**:
  - Use "Card" layouts for metrics and charts to create visual separation.
  - Standardize `st.metric` with consistent colors.
  - Custom CSS for high-contrast Plotly charts.

## 3. Component Breakdown

### 3.1 Global Theme Toggle
- **Location**: Top of Sidebar.
- **Component**: `st.toggle` or `st.radio` (horizontal).
- **Behavior**: On change, update session state and call `st.set_theme()`.

### 3.2 Refined Quant Tool Page
- **Removal**: Delete the 70+ lines of hardcoded dark-only CSS.
- **Dynamic Styling**: Use a `get_custom_css(theme)` helper to inject theme-aware styles.
- **Modern Cards**: Use `st.container` with custom CSS `border-radius` and `box-shadow` for the "OFF Filter Analysis" card.

### 3.3 Main App Unification
- Apply the same card styling to "Stock Selector" and "Stock Analysis" tabs.
- Standardize the `st.progress` bars to match the accent colors.

## 4. Success Criteria
- [ ] App defaults to Light mode on fresh load.
- [ ] Toggle to Dark mode instantly updates the entire app (including Quant Tool).
- [ ] No hardcoded dark-mode colors remain in the codebase.
- [ ] All charts adjust their background/grid colors based on the theme.
- [ ] `pytest` tests pass (logic remains intact).

## 5. Design Decisions
- **Decision 1**: Use `st.set_theme` instead of re-injecting CSS variables for core Streamlit components to ensure stability.
- **Decision 2**: Keep `quant_tool_page.py` as a separate module but remove its style independence.

## 6. Self-Review
- **Placeholder Scan**: None.
- **Consistency**: The "Modern Trading (Minimal)" style is applied globally.
- **Scope**: Focused on UI/UX and Theme management. No changes to trading logic or data fetchers.
- **Ambiguity**: Clearly defined "Light" vs "Dark" behaviors.
